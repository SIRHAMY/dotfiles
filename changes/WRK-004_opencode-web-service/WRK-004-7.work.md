# Worklog: WRK-004-7 - Profile-wide OpenCode installation

**Unit:** WRK-004-7
**Route:** PROCEED
**Status:** done

## Plan

- Add one idempotent `install-opencode` recipe that verifies the downloaded pinned official v1.18.3 installer before running it under a system-only `PATH`, then verifies its executable/version contract at `~/.opencode/bin/opencode`.
- Invoke that recipe from all three profile dependency recipes, and expose the installed path to macOS zsh without using Homebrew.
- Update the change decision/checklist and root setup instructions while preserving and not staging the root README's independent edits.

## Checks (planning bar)

- **PB-EXISTS** (problem still exists) — VERIFIED — the staged `install-opencode` recipe streams an unverified installer and lets the official script inspect the ambient `PATH`, so a global same-version OpenCode can prevent the required target from being created.
- **PB-SENSE** (still makes sense) — VERIFIED — `CHANGE.md` still names OpenCode as a fail-loud prerequisite, so `just setup` does not yet meet the user-directed installation contract.
- **PB-PLAN** (plan achieves impact, in bounds) — VERIFIED — the tagged installer source supplies a stable SHA-256, and `/usr/bin/sha256sum` (Linux) or `/usr/bin/shasum -a 256` (macOS) verify it before the fixed system-only `PATH` executes it.

## Judgment calls

- Should the recipe accept a PATH-provided OpenCode binary? → No (high confidence) — the service requires `~/.opencode/bin/opencode`, so idempotence and verification must target that executable directly.
- Should installer verification rely on a mutable endpoint? → No (high confidence) — the installer URL and committed digest both name v1.18.3, so the downloaded bytes must match the reviewed tagged source.

## Where it left off

- Completed: the pinned installer is downloaded to a private temporary file, verified against the committed tagged-source SHA-256, and run under the system-only `PATH`; the target executable/version assertion remains unchanged.
- Validation passed: `just --list`, `zsh -n`, `git diff --check`, and an installer mock that served a fake release while a same-version global OpenCode was present. The target was installed, the global executable never ran, and an idempotent rerun succeeded with curl forced to fail. `just --fmt --check` remains nonzero because it proposes pre-existing whole-file `justfile` formatting changes.
